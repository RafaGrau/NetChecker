Comprobación de conectividad

Pulse para iniciar. La aplicación procesa cada servidor individualmente: primero todos sus puertos TCP, luego los UDP.

Durante el test la barra de estado muestra el puerto en curso y la barra de progreso el avance global. Pulse para cancelar.

Tiempo de espera (Timeout)

El panel de Timeout en la barra de estado muestra el valor actual. Haga clic sobre él para seleccionar entre:

  • 500 ms — para redes LAN rápidas.
  • 1000 ms — valor intermedio recomendado.
  • 2000 ms — para redes lentas o enlaces WAN (valor por defecto).
El timeout afecta tanto a TCP como a UDP. Un valor menor acelera el test pero puede dar falsos negativos en redes con latencia alta.

Método de comprobación TCP

Para cada puerto TCP se realiza un handshake completo (SYN → SYN-ACK → ACK) usando un socket no bloqueante con connect() + select(). Si el handshake tiene éxito:

  • Se envía un byte de prueba al servicio.
  • Se espera hasta 500 ms por una respuesta (banner del servicio).
  • Los servicios que envían banner (FTP, SMTP, RDP, LDAP...) mostrarán bytes en Rx. Los que no hablan hasta que el cliente envía datos mostrarán —.
  • En Tx se cuentan los ~60 bytes del propio handshake TCP más el byte de prueba.

Método de comprobación UDP

Para cada puerto UDP se envía un byte de prueba y se espera respuesta. Con SIO_UDP_CONNRESET activado, Windows propaga los mensajes ICMP port-unreachable como errores de socket:

  • Datos recibidos → OK
  • ICMP port-unreachable → CERRADO
  • Timeout sin respuesta → SIN RESPUESTA (el puerto puede estar abierto o filtrado)

Activar / desactivar puertos

Marque o desmarque el checkbox de cada fila. Con botón derecho puede habilitar/deshabilitar grupos completos (todos, solo TCP o solo UDP).

Los cambios en los checkboxes se sincronizan entre la vista lista y la vista por pestañas.
