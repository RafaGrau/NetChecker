NetChecker – Guía de usuario

Comprobador de conectividad TCP/UDP para entornos Windows empresariales  ·  v1.0

¿Qué es NetChecker?

NetChecker es una herramienta de diagnóstico de red para administradores de sistemas Windows. Permite verificar de forma rápida si un conjunto de puertos TCP y UDP están accesibles desde el equipo local hacia uno o varios servidores de destino.

Es especialmente útil antes de desplegar roles de Active Directory, servidores de impresión o agentes SCCM/ConfigMgr, ya que incluye los perfiles de puertos estándar de cada tipo de servidor.

Requisito: La aplicación se ejecuta en Windows x64. No requiere instalación; es un único ejecutable.

Primeros pasos

  1. Inicie NetChecker.exe. Si no existe ningún archivo de configuración, se abrirá automáticamente el Editor de configuración.
  2. Añada al menos un servidor con su nombre, dirección IP y tipo (DC, PrintServer, SCCM, SCCM DP).
  3. Revise la lista de puertos de cada servidor y marque o desmarque los que desee comprobar.
  4. Pulse Aceptar para guardar la configuración.
  5. Pulse el botón Comprobar (▶) en la barra de herramientas para iniciar el test.
Consejo: La IP de origen se detecta automáticamente y se muestra en la barra de estado. No se almacena en el archivo de configuración.

Configuración de servidores

El editor de configuración se abre pulsando el icono ✏ de la barra de herramientas, o automáticamente la primera vez que se inicia la aplicación.

Agregar un servidor

  1. Introduzca el Nombre del servidor en el campo de texto.
  2. Introduzca la IP de destino usando el control de máscara (separado por puntos automáticamente).
  3. Seleccione el Tipo de servidor en el desplegable: DC, PrintServer, SCCM o SCCM DP.
  4. Pulse Agregar. Se crea una pestaña con los puertos por defecto de ese tipo.
Modificar un servidor existente: haga clic en su pestaña. Los campos se rellenan con sus datos. Modifique lo que necesite y pulse Actualizar.

Gestionar puertos

La cuadrícula de la pestaña muestra los puertos TCP y UDP del servidor. Puede:

  • Agregar puerto: introduzca el número, protocolo y descripción en la parte inferior y pulse Agregar puerto. La descripción se rellena automáticamente si es un puerto conocido.
  • Modificar puerto: seleccione la fila, edite los campos y pulse Actualizar.
  • Eliminar puerto: seleccione la fila y pulse Eliminar puerto.
  • Ordenar: haga clic en la cabecera Puerto o Protocolo para ordenar ascendente/descendente (▲/▼).

Importar / Exportar CSV

Los botones Importar CSV y Exportar CSV permiten intercambiar configuraciones de servidores en formato de texto plano.

Formato del archivo:

hostname,ip,type,ports (port/proto)
DC01,10.0.0.1,DC,88/TCP,88/UDP,389/TCP,445/TCP,636/TCP
SRV-PRINT,10.0.0.5,PrintServer,80/TCP,443/TCP,515/TCP,631/TCP,9100/TCP
  • Columna type: DC, PrintServer, SCCM, SCCM_DP.
  • Al importar, si el puerto es conocido se rellena la descripción automáticamente.
  • Al importar, los servidores se añaden a los existentes (no se sobreescriben).

Comprobación de conectividad

Pulse el botón Comprobar (▶). La aplicación procesará cada servidor de forma completa (primero todos sus puertos TCP, luego los UDP) antes de pasar al siguiente.

Durante el test:

  • La barra de progreso muestra el avance global.
  • La barra de estado indica el puerto que se está comprobando en ese momento.
  • Cada fila se actualiza en tiempo real conforme se obtiene el resultado.

Puede pulsar el botón de nuevo (ahora ■) para detener el test en cualquier momento.

Los puertos desmarcados (checkbox vacío) no se comprueban y se muestran con estado Deshabilitado.

Activar / desactivar puertos individualmente

Haga clic en el checkbox de una fila para marcar o desmarcar ese puerto. También puede usar el menú contextual (botón derecho) sobre la lista para habilitar o deshabilitar grupos completos:

  • Habilitar / deshabilitar todos los puertos del servidor.
  • Habilitar / deshabilitar solo los puertos TCP.
  • Habilitar / deshabilitar solo los puertos UDP.
Nota: Los cambios realizados en la vista lista y en la vista por pestañas se sincronizan automáticamente.

Modos de vista

El botón de vista en la barra de herramientas alterna entre dos modos:

🗂 Vista en lista

Todos los servidores aparecen en una única lista agrupada por servidor. Es el modo por defecto y el más compacto para comparar resultados.

📑 Vista por pestañas

Cada servidor ocupa una pestaña independiente. Ideal cuando hay muchos puertos o se quiere analizar un servidor concreto sin distracciones.

El icono del botón cambia según el modo activo: muestra el modo al que se puede cambiar.

Informe HTML

Pulse el icono Guardar HTML para exportar los resultados. El informe incluye:

  • IP de origen detectada, fecha y hora de generación.
  • Una tabla por servidor con los puertos comprobados (los deshabilitados no aparecen).
  • Puertos TCP primero, luego UDP dentro de cada tabla.
  • Columnas: Puerto, Protocolo, Descripción, Estado, Latencia, Tx, Rx.
  • Colores de estado coherentes con la vista de la aplicación.

El nombre del archivo sugerido tiene el formato NetChecker_AAAAMMDD_HHMM.html. El informe es autocontenido y puede abrirse en cualquier navegador.

Estados de puerto

Estado Significado
OK Conexión establecida correctamente. El puerto está abierto y accesible.
CERRADO El destino rechazó la conexión (TCP RST o ICMP port-unreachable para UDP). El servicio no está escuchando o un firewall lo rechaza activamente.
SIN RESPUESTA Solo UDP. No se recibió respuesta en el tiempo de espera. El puerto puede estar abierto (el servicio no responde al probe) o filtrado por un firewall silencioso.
PENDIENTE El puerto aún no ha sido comprobado en esta sesión.
— Puerto deshabilitado. No se comprueba y no aparece en el informe HTML.
Sobre UDP: A diferencia de TCP, UDP no tiene establecimiento de conexión. El estado SIN RESPUESTA es habitual en puertos UDP correctamente configurados que no devuelven datos al probe. Un resultado OK solo se obtiene si el servicio responde con datos.

Archivo de configuración

La configuración se almacena en NetChecker.config en formato XML junto al ejecutable. Puede editarse manualmente con cualquier editor de texto.

<?xml version="1.0" encoding="UTF-8"?>
<NetCheckerConfig version="3">
  <Destinations>
    <Destination name="DC01" ip="10.0.0.1" type="DC">
      <Ports>
        <Port port="88"  protocol="TCP" enabled="1"/>
        <Port port="389" protocol="TCP" enabled="1"/>
        <Port port="445" protocol="TCP" enabled="0"/>
      </Ports>
    </Destination>
  </Destinations>
</NetCheckerConfig>
  • El atributo enabled="0" indica que el puerto está deshabilitado.
  • La descripción de cada puerto se resuelve automáticamente al cargar; no se guarda en el XML.
  • La IP de origen tampoco se guarda; se detecta en cada inicio.
Tipos de servidor válidos: DC, PrintServer, SCCM, SCCM_DP.
